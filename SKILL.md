# OpenHands Agent

Full-stack coding agent using [OpenHands](https://github.com/All-Hands-AI/OpenHands) — an AI software engineer that operates in a sandboxed environment.

## Commands

### Start OpenHands server

```bash
python3 openhands_agent.py start
python3 openhands_agent.py start --port 3001
python3 openhands_agent.py start --local
```

Starts the OpenHands server. Uses Docker by default; `--local` runs without Docker.

### Submit a coding task

```bash
python3 openhands_agent.py task "Build a REST API for a todo app with FastAPI"
python3 openhands_agent.py task "Fix the login bug in app.py" --workspace /path/to/project
python3 openhands_agent.py task "Add unit tests for the user module" --timeout 600
```

Submits a natural language coding task to OpenHands. Results are saved to the workspace.

### Check task status

```bash
python3 openhands_agent.py status
python3 openhands_agent.py status --task-id abc123
```

Shows the current state of running tasks and sandbox status.

### Manage workspace

```bash
python3 openhands_agent.py workspace list
python3 openhands_agent.py workspace show <task-id>
python3 openhands_agent.py workspace pull <task-id> /local/path
python3 openhands_agent.py workspace clean
```

View, extract, or clean up sandbox workspace files.

### Review code changes

```bash
python3 openhands_agent.py review <task-id>
python3 openhands_agent.py review <task-id> --diff
python3 openhands_agent.py review <task-id> --apply /target/repo
```

Review code generated by OpenHands. Optionally show a diff or apply to a local repo.

### Configure settings

```bash
python3 openhands_agent.py config
python3 openhands_agent.py config --model claude-sonnet-4-20250514
python3 openhands_agent.py config --api-key sk-...
python3 openhands_agent.py config --sandbox docker
```

View or update OpenHands configuration (model, API keys, sandbox type).

## Workflow

### Full development cycle

1. Start server: `python3 openhands_agent.py start`
2. Submit task: `python3 openhands_agent.py task "implement feature X"`
3. Monitor: `python3 openhands_agent.py status`
4. Review: `python3 openhands_agent.py review <task-id> --diff`
5. Apply: `python3 openhands_agent.py review <task-id> --apply ./my-project`

### Configuration

Set your API key before first use:
```bash
export ANTHROPIC_API_KEY=sk-...
# or
export OPENAI_API_KEY=sk-...
```

OpenHands uses LiteLLM internally, so any supported provider works.

## Notes

- OpenHands runs code in a sandbox — changes don't affect your system until applied
- Docker mode provides stronger isolation but requires Docker installed
- Local mode runs without Docker but with reduced sandboxing
- Tasks have a default timeout of 300 seconds; increase for complex tasks
- The workspace persists task outputs for review before applying
